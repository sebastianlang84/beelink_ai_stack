# Rolling-Window Semantik (Short-term vs Long-term)


## Status

Policy-/Doku-Notiz. Diese Datei ist ein Platzhalter, weil andere Docs im Repo darauf verweisen.

## Idee (kurz)

- *Short-term Inputs*: aktuelle Transkripte innerhalb eines definierten Cutoffs (z.B. letzte N Tage).
- *Long-term Inputs*: ältere, bereits analysierte Artefakte (History-Bundles) als Kontext.

Hinweis: Konkrete Cutoff-Regeln sind topic-/config-abhängig und müssen aus der jeweiligen Config/Implementierung abgeleitet werden.
